Arquitectura de contenido: ¿cómo organizamos todo lo que Colmena hace?

Colmena tiene 12+ capacidades. Tres formas de presentarlas sin abrumar.

A

"Capability Groups" — Agrupado por función

Secciones: Hero con hexágono → Value props agrupadas en 4 categorías (Governance, Multi-Agent, Trust & Audit, Operator Tools) → Diagrama de arquitectura animado → "Colmena vs Auto-Mode" → Use cases → Install → CTA.

Fortaleza: El usuario escanea categorías y encuentra lo que le importa. Riesgo: Puede sentirse como un "feature list".

B

"The Mission Flow" — Contado desde la experiencia del operador

Secciones: Hero → "The Problem" (caos multi-agente) → "How Colmena Works" en 4 pasos visuales (Define → Spawn → Execute → Review) → Value props emergen naturalmente en cada paso → "Your First Mission" (quick start integrado) → Use cases → CTA.

Fortaleza: Narrativa fluida, el usuario "vive" el producto. Riesgo: Algunas features quedan escondidas si no encajan en el flow.

C

"The Dashboard" — Una sola pantalla de alto impacto

Secciones: Hero con hexágono + tagline → Grid de 6-8 cards visuales (una por capacidad clave, con ícono + una línea) → Diagrama de arquitectura central → "Start in 30 seconds" → CTA. Todo en 2-3 pantallas de scroll.

Fortaleza: Máximo impacto visual, ideal para mobile y atención corta. Riesgo: No profundiza lo suficiente para el que quiere evaluar en serio.

Mi recomendación

Combinar A + B: Usar "The Mission Flow" como esqueleto narrativo principal, pero agrupar las value props en categorías visuales dentro de cada paso del flow. Así el que escanea encuentra categorías claras, y el que lee entiende la historia completa. La estructura sería:

HERO → THE PROBLEM → HOW IT WORKS (4 steps) → VALUE PROPS (grouped) → COLmena VS AUTO-MODE → USE CASES → QUICK START → CTA

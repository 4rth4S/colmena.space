How it works

One mission, one cycle, full accountability.

1
Define

Write a YAML manifest. Pick roles, scope paths, set a TTL. One file defines the mission.

mission.yaml
2
Spawn

One command spawns the squad. Each agent gets its role, its tools, its boundaries.

colmena mission spawn
3
Execute

Agents work in parallel. Every tool call evaluated in <15ms. Audit.log on every decision.

ALLOW · ASK · BLOCK
4
Review

Auditor scores every artifact. ELO adjusts. Trust is earned — not declared.

QPC scoring
Define → Spawn → Execute → Review → repeat, with better trust each cycle

Mission Manifest — insertado en "How It Works" (paso 1: Define), fondo oscuro
The plug & play layer

One YAML file. One command. Full mission.

Define roles, scope, bash patterns, TTL, and acceptance criteria in a manifest.
Colmena handles validation, agent spawning, delegations, and the review cycle.

mission: review-auth-module
pattern: code-review-cycle
ttl_hours: 4

roles:
  - name: developer
    task: harden auth
    scope:
      paths: [src/auth/]
      bash_patterns:
        - ^cargo (test|build)
  - name: auditor
    task: review + score
✓ Auto-validates schema + security
✓ Generates role prompts + subagent files
✓ Creates scoped delegations with mandatory TTL
✓ Wires bash_patterns into runtime overrides
✓ Activates mission gate automatically
✓ --dry-run to preview before executing

Already running agents? mission init --from-history reads your audit.log and proposes a manifest skeleton from your actual usage patterns.

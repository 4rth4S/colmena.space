How it works

One YAML file. One command. Full accountability.

1. Define

Write a YAML manifest. Pick roles, scope paths, bash patterns, set a TTL.

mission: review-auth
pattern: code-review-cycle
roles:
  - name: developer
    scope:
      paths: [src/auth/]
  - name: auditor

2. Spawn

One command spawns the squad. Each agent gets its role, its tools, its boundaries.

colmena mission spawn
3. Execute

Agents work in parallel. Every tool call evaluated in <15ms. Audit.log on every decision.

ALLOW · ASK · BLOCK
4. Review

Auditor scores every artifact. ELO adjusts. Trust is earned — not declared.

QPC scoring
Already running agents? mission init --from-history reads audit.log and proposes a manifest from your patterns.
